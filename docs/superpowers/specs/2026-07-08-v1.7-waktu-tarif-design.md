## 1. Tujuan

Tiap rute hasil menampilkan **estimasi waktu tempuh** (menit) dan **tarif** (Rupiah)
di baris ringkasan, di samping `transfer · halte` yang sudah ada. Menambah informasi
keputusan buat user tanpa mengubah algoritma pencarian rute.

## 2. Keputusan terkunci (DECIDED 2026-07-08)

1. **DECIDED — Tampilan-saja.** Router inti tetap Pareto 2D `(transfer, halte)`.
   Waktu & tarif **dihitung lalu ditampilkan** per rute hasil, BUKAN dimensi
   pencarian. (v1.8: pertimbangkan masuk Pareto — diskusi terpisah, di luar scope ini.)
2. **DECIDED — Tarif akurat (aturan TJ nyata).** BRT Rp3.500 flat sekali bayar;
   transfer antar-koridor dalam sistem (pindah peron / transfer resmi) GRATIS;
   Mikrotrans Rp0; Royaltrans/Transjabodetabek flat per-naik. Lihat §5.
3. **DECIDED — Waktu = ride saja.** Cuma waktu di dalam bus (selisih `arrival_time`
   antar-halte). TANPA waktu tunggu & TANPA menit jalan-kaki di angka headline.
   Deterministik, tak perlu asumsi kedatangan acak.

## 3. Prinsip arsitektur (jaga pola v1.6)

- **build-data.py hitung → data.json simpan → UI surface.** Sama seperti v1.6
  memindahkan `xdist`. Router (`router.js` search) tidak diubah.
- **`route.py` = oracle.** Ikut hitung waktu & tarif dengan rumus identik supaya
  `test-router.js` parity tetap sah dan angka Python == angka JS.
- **Helper murni, tanpa DOM.** Perhitungan per-rute di modul pure (Node + browser),
  dipakai ulang di dua call-site render.

## 4. Perubahan data — `build-data.py`

Tambah 2 field ke `data.json`. Feed ini berbasis `frequencies.txt`, jadi
`stop_times.txt` memuat jam untuk SATU trip representatif per pola; **selisih**
`arrival_time` antar-halte berurutan = durasi tempuh segmen (valid sebagai ride time).

### 4a. `etime` — waktu tempuh per edge (detik)

Cermin struktur `edges` (`{routeIdx:{stopIdx:{nextStopIdx: detik}}}`):

```
etime[routeIdx][stopIdx][nextStopIdx] = median durasi tempuh (detik)
```

Algoritma:
1. Untuk tiap trip, dari `stop_times.txt` ambil urutan `(stop_sequence, stop_id, arrival_time)`.
2. Parse `arrival_time` "HH:MM:SS" → detik (boleh >24:00:00; GTFS izinkan).
3. Per pasangan berurutan `(a,b)`: `delta = arr_b − arr_a`.
4. Kumpulkan delta per edge berarah `(routeIdx, aIdx, bIdx)` lintas semua trip route itu.
5. Simpan **median** delta (robust vs outlier / trip aneh).
6. **Fallback** kalau `delta ≤ 0` atau tak ada data valid:
   `dist_m / BUS_SPEED_MS`, `dist_m` dari haversine lat/lon (sudah ada di build).
   `# ponytail: BUS_SPEED_MS knob dunia-nyata; default ~6 m/s (~21 km/h) TJ padat.`

### 4b. `fare` — tarif per route

```
fare[routeIdx] = [price_int, fareClass_str]   # mis. [3500, "FP"], [0, "GR"], [20000, "PP"]
```

Bangun dari:
- `fare_rules.txt`: `route_id → fare_id` (satu fare_id per route; kalau route punya
  banyak, ambil yang pertama — feed ini 1:1).
- `fare_attributes.txt`: `fare_id → (price, ...)`.
- Route tanpa entri fare_rules → default `[0, "?"]` dan `print` warning (jangan diam).

Kelas fare di feed ini:

| fare_id | price | arti | aturan tarif |
|---|---|---|---|
| FP, FP2 | 3500 | BRT flat | sekali per sesi BRT (lihat §5) |
| GR | 0 | Mikrotrans (JAK) | selalu gratis |
| PP | 20000 | Royaltrans/Transjabodetabek | flat per-naik |
| PP2 | 25000 | idem tingkat lebih tinggi | flat per-naik |
| PP3 | 35000 | idem tingkat tertinggi | flat per-naik |

## 5. Perhitungan per-rute (helper murni)

Modul baru **`web/cost.js`** (pola sama `legs.js`: UMD, `window.Cost` + Node export).
Fungsi `routeCost(path, data) → { secs, fare }`, jalan di atas `path` hasil router
(5-tuple: `kind, stop, route, xtype, xdist`).

> Alternatif lokasi: taruh di `legs.js` sebagai fungsi kedua. Pilih `cost.js`
> terpisah biar file fokus. Yang penting: pure, teruji, dipakai 2 call-site.

### 5a. Waktu (`secs`)

Jumlahkan `etime` tiap segmen `ride`:
- Telusuri `path`. Untuk tiap transisi ke stop berikut DALAM route yang sama
  (langkah `ride`), tambahkan `etime[route][prevStop][thisStop]`.
- Konversi tampil: `Math.round(secs/60)` → `~N mnt`.

### 5b. Tarif (`fare`) — aturan TJ (DECIDED akurat)

Telusuri leg per leg (satu leg = satu `take` = satu naik-bus). Lacak state
`brtPaid` (boolean: sesi BRT berbayar sedang aktif).

Untuk tiap leg dengan `fareClass` dan `xtype` (jenis transfer MASUK leg ini;
leg pertama `xtype` = null):

```
GR            → +0.            (Mikrotrans; tidak mengubah brtPaid)
PP/PP2/PP3    → +price.        (layanan terpisah, tap sendiri; tidak set brtPaid)
FP/FP2 (BRT)  → sesi BRT:
   masuk lewat transfer dalam-sistem (xtype ∈ {null(awal), "s", "o"})
       DAN brtPaid == true  → +0   (transfer koridor gratis)
   selain itu (leg BRT pertama, ATAU masuk lewat "w" jalan-kaki keluar sistem)
       → +price; set brtPaid = true
```

Aturan reset: transfer **`w`** (jalan kaki, keluar sistem) → sesi BRT putus →
leg BRT berikutnya bayar lagi. Transfer **`s`** (pindah peron) / **`o`** (resmi)
→ tetap dalam sistem, sesi lanjut.

Catatan penting: transfer **nama-sama** (halte fisik sama) sudah di-tag **`"s"`**
oleh router (`route.py:217` `targets = [(s2, "s", 0) for s2 in name_stops[...]]`;
cermin di `router.js`). Jadi `xtype` cukup 3 nilai `s/o/w` + `null` (leg awal) —
tak ada kategori "nama-sama tanpa xtype". `leg.xtype` hasil `pathToLegs` = jenis
transfer yang MASUK leg itu, tersedia langsung.

`# ponytail: apakah "w" memutus sesi tarif = knob. Default: ya (keluar gate = tap ulang).`
`# ponytail: window transfer TJ 3 jam (transfer_duration=10800). Ride-saja tak lacak`
`#           jam dinding; asumsi dalam window (trip pendek). Cukup untuk estimasi.`

Tampil: `"Rp" + fare.toLocaleString("id-ID")` → `Rp3.500`. Kalau `fare == 0` →
`"Gratis"`.

Tambah fungsi `route_cost(path, etime, fare) → (secs, fare_rupiah)` dengan rumus
IDENTIK §5. `load()` bangun `etime` + `fare` dari GTFS (rumus sama build-data).
`render()` cetak baris `~N mnt · RpX` supaya bisa dibanding manual vs UI.

**Selftest `_selftest()` tambah kasus tarif** (aturan TJ):
- Rute BRT 2-koridor, transfer tipe `s` → total **Rp3.500** (bukan 7.000).
- Rute BRT + BRT via transfer `w` (jalan kaki) → **Rp7.000** (sesi putus).
- Rute BRT + Mikrotrans (GR) → **Rp3.500** (GR gratis).
- Rute Royaltrans (PP) tunggal → **Rp20.000**.
- Waktu: edge dummy dengan delta arrival dikenal → `secs` = jumlah delta.

`test-router.js` (JS) tambah kasus cermin biar parity Python==JS terjaga.

  const c = window.Cost.routeCost(routeOptions[0].path, data);

di `app.js` atau ekspor dari `cost.js`.
