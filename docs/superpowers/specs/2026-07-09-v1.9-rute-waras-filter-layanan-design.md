**Tanggal:** 2026-07-09 · **Status:** Final (disetujui Reza, sesi CC)
**Menggantikan sebagian:** v1.8 (tab "Waktu tercepat" & "Minim jalan-kaki" dihapus)

## 1. Masalah (temuan Reza di v1.8 live, diverifikasi di kode)

1. **Naik di halte lain dari yang dipilih.** Pilih naik "Simpang Kuningan", hasil
   menyuruh naik di "Lintas Atas Kuningan Mampang". Akar: sebelum naik bus pertama,
   router boleh loncat via link jalan kaki `w` gratis (`shortestGoal` seed
   `route:null` lalu blok transfer tanpa pembatasan; `web/router.js` ±285–300).
2. **Leg naik-turun di halte sama** ("Naik: 1-1 Blok M · Turun: 1-1 Blok M").
   Akar: cek arrival terjadi persis setelah langkah "take" tanpa satu edge ride pun.
3. **Tab "Waktu tercepat" gak waras** (6 transfer + 🚶742m "≈18 mnt"). Akar: cost
   goal `time` = detik ride SAJA — transfer & jalan kaki = 0 detik (asumsi v1.7
   "waktu=ride saja" dibuat saat waktu masih tampilan; rusak begitu jadi dimensi optimasi).
4. **"Tarif terendah" ≈ "Minim jalan kaki" sering identik** (mayoritas rute
   Rp3.500 flat → banyak seri → tie-break jatuh ke rute sama).

## 2. Keputusan (DECIDED 2026-07-09)

- **D1 Origin lock:** dari state belum-naik-bus (`route === null`), transfer keluar
  hanya `s` (pindah peron nama-sama). Link `w`/`o` DILARANG, **kecuali** targetnya
  halte tujuan → langsung selesai (rute jalan-kaki-saja, tampil blok transfer sesuai
  jenis 🚶/🔗, tanpa leg bus).
  Berlaku SEMUA pencarian: 3 goal-Dijkstra + Pareto (kolam Kejutan).
- **D2 Aturan ride-minimal:** bus hanya "dinaiki" kalau minimal 1 edge ride ditempuh
  di leg itu. Menutup leg-0-halte di ujung DAN take beruntun di tengah rute.
  Kalau tujuan tercapai via transfer/jalan dari halte terakhir → rute selesai di situ,
  langkah penutup tampil sebagai blok transfer (🚶/🔗/↔️) + 🏁, BUKAN leg bus.
- **D3 Tab baru 2×2:** 💰 Tarif terendah · 🧘 Paling simpel · 📏 Jarak terpendek ·
  🎲 Kejutan (tetap beta v1.8). "Waktu tercepat" & "Minim jalan-kaki" DIHAPUS.
- **D4 Paling simpel = weighted cost lama:** `transfer × 8 + halte` (WEIGHT v1.1.1,
  teruji). BUKAN minim-transfer-murni — itu membangkitkan bug muter v1.1.1
  (0tf/16-halte loop L13E menang vs 1tf/2-halte). Use case acuan: Simpang
  Kuningan→CSW 1 = naik L13E ke Patra, pindah peron, L13E balik (1 transfer).
- **D5 Jarak terpendek:** cost = Σ `dist` ride (meter) + Σ `xdist` transfer `w`;
  `s`/`o` = 0 m. **Butuh field data baru** (§3) — `dist` antar halte belum ada di
  data.json (baru `xdist` di `xfer`). Catatan jujur: rute lebih-banyak-transfer boleh
  menang kalau memang lebih pendek — label tidak bohong, transfer terlihat di kartu.
- **D6 Menit kartu = estimasi kasar, tampilan-saja:**
  `detik = ride_secs(etime v1.7) + TRANSFER_WAIT_S × transfer + walk_m ÷ WALK_SPEED_MS`.
  `TRANSFER_WAIT_S = 240`, `WALK_SPEED_MS = 1.4` — tuning knobs, konstanta bernama.
  Berlaku semua kartu (angka "18 mnt ride-only" yang menyesatkan hilang).
- **D7 Filter layanan** (§5): checkbox 8 kelas, batasan keras 3 lapisan
  (saran halte, router, otomatis semua leg). Default semua ON, persist localStorage.
- **D8 Parity:** semua aturan cost/lock/tie-break dicermin `route.py`; cost & tie-break
  INTEGER SAJA (meter/detik/hitungan bulat — jangan float, beda pembulatan
  Python vs JS pecahkan parity di kasus seri).

## 3. Data (`build-data.py`) — 1 field baru

- `dist` = `{routeIdx: {stopIdx: {nextStopIdx: meter}}}` (bentuk sama `etime`).
  Haversine antar halte berurutan per trayek, **dibulatkan integer meter di Python**
  (satu otoritas — JS cuma baca, parity aman). Reuse `_haversine_m` yang sudah ada.
- `test_build.py`: section baru — `dist` ada utk semua edge di `edges`, semua int > 0,
  spot-check 1–2 edge nilai wajar (ratusan meter, bukan ribuan km).
- Field lain TIDAK berubah.

## 4. Router (`web/router.js` + `route.py`, parity)

### 4a. Perubahan `shortestGoal` / `find_goals`
- Goal id: `fare` (tetap) · `simple` (baru, ganti `walk`) · `dist` (baru, ganti `time`).
- **Origin lock (D1):** saat `route === null`, target transfer hanya `s`;
  `w`/`o` hanya jika target ∈ `dests` → return rute jalan-kaki-saja.
- **Ride-minimal (D2):** state tambah flag `ridden` (sudah tempuh ≥1 edge sejak
  take terakhir). "Take" berikutnya hanya boleh dari `route === null` atau `ridden`.
  Saat ekspansi transfer ke `s2 ∈ dests`: JANGAN board — emit langkah penutup
  (jenis transfer apa adanya) lalu selesai.
- **Cost per goal:**
  - `fare`: rumus v1.7 §5b, state `(stop, route, brtPaid)` — tak berubah.
  - `simple`: take menambah `8` (WEIGHT), ride menambah `1`. (= `tr×8 + st` aditif.)
  - `dist`: ride menambah `dist[route][from][to]`; take `w` menambah `xdist`; `s`/`o` 0.
- **Label & tie-break seragam (semua goal):** banding urut
  `(cost, walkM, tr, st, seq)` — `walkM` = akumulasi meter jalan kaki, dimensi
  dominance ikut jadi 4D `(cost, walkM, tr, st)`. `seq` deterministik karena target
  transfer & daftar trayek tetap di-sort (mekanisme v1.8 dipertahankan).
  Ini sekaligus jawaban temuan #4: seri tarif dipecah oleh jalan-kaki-terdikit dulu.
- `MAX_GOAL_TRANSFERS = 6` tetap.
- **Filter (D7):** `shortestGoal`/Pareto terima set kelas layanan; trayek dengan
  `rtype` di luar set DI-SKIP saat ekspansi (ride & take). Default = semua kelas.
- Pareto v1.5 (kolam Kejutan) ikut D1+D2+filter; model biayanya sendiri tak berubah.

## 5. Filter layanan (UI + lapisan)

- **UI:** baris chips checkbox 8 kelas di atas form cari, dari nilai `rtype` nyata:
  BRT · Angkutan Umum Integrasi · Mikrotrans · Transjabodetabek · Royaltrans ·
  Rusun · Shuttle · Bus Wisata. Default semua ON. Persist `localStorage` (key ikut
  pola penamaan storage app yang ada).
- **Lapisan 1 — saran halte:** autocomplete + urut-📍 hanya tampilkan halte yang
  dilayani ≥1 trayek lolos filter. (Precompute set halte-terlayani dari `edges`+`rtype`
  saat filter berubah.)
- **Lapisan 2 — router:** §4a. Otomatis semua leg patuh (lapisan 3 gratis).
- **Guards (wajib):**
  1. Filter ≠ semua-ON → indikator terlihat di area hasil (mis. "Filter: BRT, Mikrotrans").
  2. Tidak ada rute → pesan jelas: "Tidak ada rute dengan filter aktif — coba longgarkan filter."
  3. Halte asal/tujuan sudah terpilih lalu filter berubah → validasi saat cari;
     kalau halte tak lagi terlayani, error menyebut nama halte + filternya.
- Parity: `route.py` CLI terima daftar kelas opsional (arg), default semua.
