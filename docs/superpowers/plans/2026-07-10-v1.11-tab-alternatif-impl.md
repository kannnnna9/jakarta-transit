# v1.11 "Tab Alternatif" — Plan Implementasi (Codex)

**Spec (sumber kebenaran, baca penuh):** `docs/superpowers/specs/2026-07-10-v1.11-tab-alternatif-design.md`
**Goal:** ganti tab "🎲 Kejutan" (random) → "🔀 Alternatif" (deterministik). Tab ini boleh jalan kaki lebih jauh (`ACCESS_M=400m`) dari origin ke halte tetangga → nemu rute nyata yang router lama buta (Simpang Kuningan → jalan 297m → Underpass → route 6 one-seat Ragunan, 0 transfer).
**Aturan mati:** 3 tab lama (Tarif/Simpel/Jarak) BYTE-IDENTIK, nol regresi. Alternatif = pencarian TERPISAH. Parity Python↔JS wajib, integer meter, tie-break deterministik.

## Anchor kode (terverifikasi @ commit b1b7e53)

### `route.py`
- Konstanta baris ~25-28 (`WALK_M=150`, `STOP_M=40`, `DIST_TRANSFER_M=200`) → tambah `ACCESS_M=400`.
- REUSE (jangan tulis ulang): `_haversine_m`, `_stop_walk_m(a,b,stop_lat,stop_lon)`, `_allowed(route,rtype,allowed)`.
- Model referensi: `find_goal(...)` baris ~435. Return `(tr, st, path, cost)`. Data tuple: 0 stop_name·1 name_stops·2 rname·3 ride·4 routes_at·5 xfer·6 etime·7 dist·8 fare_table·9 rtype·10 stop_lat·11 stop_lon. Path tuple: `("board",s,None,None,None)` / `("ride",nxt,route,None,None)`.
- Fungsi baru:
  - `find_alternative(origin, dest, data, goals_result, radius=400, allowed={"BRT"})` → `(tr, st, path)` atau `None`.
  - `_access_stops(name_substr, data, radius)` → `{stop_id: walk_m}` (semua halte radius ACCESS_M dari matched-name origin).
  - `_path_signature(path)` → signature urutan route (deteksi duplikat vs 3 goal).
- Alur find_alternative (spec §5): seed = `_access_stops`, cost/walk awal = jarak jalan, goal="simple" → kumpul Pareto → Guard A buang `stops > min_stops×1.5` → buang path == salah satu `goals_result` value → pilih 1 (min transfer → tie-break etime → tie-break stops) → return/None.
- Step jalan akses di path: tambah jenis step "access" yang bisa dirender "🚶 jalan Nm"; JANGAN pecahin rendering 3 tab lama.
- Integrasi main/endpoint: `find_goals(...)` lalu `find_alternative(...)`, ikutkan `alternative` di output.
