# v1.7 Waktu Tarif Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.7.0 with route summary estimates for ride time and fare without changing Pareto route search.

**Architecture:** `build-data.py` adds schedule-derived `etime` and route `fare` to `web/data.json`. Pure helpers in `web/cost.js` calculate `{ secs, fare }` from an existing router path; `route.py` mirrors the same rules as the parity oracle; `web/app.js` only renders the extra summary text.

**Tech Stack:** Python stdlib, browser JavaScript UMD modules, Node assert tests, existing GTFS cache.

## Global Constraints

- Router Pareto dimensions stay `(transfer, halte)`; time and fare are display-only.
- Time is ride-only from `stop_times.txt` arrival deltas; no wait time and no walking minutes in the headline.
- Fare rules: BRT FP/FP2 charges once per in-system session, `s`/`o` transfer stays in session, `w` resets BRT session, GR is free, PP/PP2/PP3 charge per boarding.
- Keep web paths relative for GitHub Pages subpath `/jakarta-transit/`.
- No new dependencies.
